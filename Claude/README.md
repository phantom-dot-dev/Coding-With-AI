Welcome to the master tutorial for **Claude Code**—Anthropic's powerful command-line coding agent loop.

Claude Code doesn’t just suggest code; it analyzes entire repositories, runs terminal commands, orchestrates sub-agents, and actively fixes bugs using an advanced **agentic loop** (gather context $\rightarrow$ take action $\rightarrow$ verify results).

---

## 1. Models Selection, Usages, & Remaining Tokens

Claude Code defaults to Anthropic’s flagship developer models (like the Claude 3.5/4.0 suites). Because it runs an agentic loop that executes multiple steps sequentially, managing your token footprint is critical.

* **View current status:** Type `/status` in the interactive CLI to view the active model, current session cost, and a breakdown of tokens used.
* **Token efficiency:** Anthropic features automatic prompt caching. Repeatedly referencing the same codebase inside a continuous session results in a heavily discounted rate for cached input tokens.

```bash
# Check current session token usage and active configuration
/status

```

## 2. Effort

For advanced mathematical reasoning, architectural design, or resolving deep-seated logic bugs, you can scale Claude's internal reasoning budget (Effort).

* **Behavior:** Increasing effort commands Claude to generate deeper internal chains of thought before executing tools or file mutations.
* **Commands:** Change this setting inline using `/effort`. Available parameters are `low`, `medium`, and `high`.

```bash
# Force high reasoning/thinking budget for a massive refactor
/effort high

```

## 3. Using Claude In VS Code & the Extension

Claude Code is a terminal-first binary, but it integrates seamlessly with Visual Studio Code.

### The Integrated Terminal Workflow

Simply open VS Code's integrated terminal (`Ctrl + \``) and launch the binary by typing `claude`. It automatically respects your active workspace directory and maps code paths directly to the files you see on your screen.

### The VS Code Extension

For a richer GUI experience, install the official **Claude Code Extension** from the marketplace. This adds:

* **Visual Diffs:** Review edits side-by-side inside your editor before committing them.
* **Inline Comments & Previews:** Seamlessly accept or reject individual blocks of code generated by the agent.

## 4. Shell Mode

Claude is fully aware of your local environment. Any command you can execute in your native terminal can be executed by Claude via **Shell Mode**.

* **Automatic Loop:** Claude will write code, invoke your shell to run a test command, parse the error output, and rewrite the code automatically until it passes.
* **Direct Execution:** You can also run commands yourself directly inside the interactive prompt.

```bash
# Tell Claude to build, run tests, and self-correct on failures
claude "Run npm run test and fix any typescript errors that crop up"

```

## 5. Modes

Claude Code has distinct execution safety settings called **Permission Modes**. You can dynamically switch modes to balance raw development speed with human-in-the-loop oversight:

| Mode | Trigger Command | Behavior |
| --- | --- | --- |
| **Standard Mode** | *Default* | Claude will ask your permission before writing to a file or running a write/execute terminal command. |
| **Auto-Edit (YOLO)** | Passed via startup flag | Claude applies code modifications and file mutations instantly without asking for approval. |
| **Plan Mode** | `/mode plan` or `Shift + Tab` | Swaps Claude into a strictly read-only state for deep analysis (detailed in Section 11). |

## 6. Settings & Permissions

You have complete control over what files, web domains, and tools Claude can access.

* **Interactive Menu:** Run `/settings` inside the CLI to configure globally trusted terminal commands, toggle auto-updating, and adjust safety thresholds.
* **Command Whitelisting:** You can configure Claude to run commands like `git` or `npm test` automatically while demanding strict manual confirmation for commands like `rm` or `curl`.

## 7. Context, Tokens, & Clearing

As you build features, your context window fills up with file data, terminal history, and compiler errors.

* **View active context:** Type `/context` to see a structured list of everything loaded into Claude's memory.
* **Compacting:** Claude compacts old conversational histories automatically, but instructions from early on can fade.
* **Clearing the buffer:** Use `/clear` to flush the conversational history while retaining pinned files, or `/reset` to completely wipe the slate.

```bash
# Clear the current chat context to drop accumulated memory overhead
/clear

```

## 8. Global Sessions

Claude Code allows you to easily switch tasks without losing your place. You can leave a complex debugging thread running in the background and attach to it later.

```bash
# Spin up an isolated global session named after a specific feature branch
claude --session database-migration

# Re-attach to that running workspace session later from any terminal pane
claude --attach database-migration

```

## 9. CLAUDE.md File & `/init`

`CLAUDE.md` acts as a **persistent system prompt** for your repository. It informs Claude about your tech stack, build scripts, styling conventions, and architectural boundaries.

* **The Command:** Run `/init` in your project root.
* **The Interview:** Claude will open an interactive questionnaire asking about your setup. It then compiles a custom `CLAUDE.md` and populates automated development scripts.

```markdown
<!-- Sample CLAUDE.md generated in your root directory -->
# Project Guidelines
- Build command: `npm run build`
- Test command: `vitest run`
- Code Style: Strict functional TypeScript, no `any` allowed.

```

## 10. Plan Mode

When you are about to execute a sweeping structural change, you want to inspect the blueprint before Claude touches a single line of production code.

* **Activation:** Type `/mode plan` or prepend your request with a plan mandate.
* **Behavior:** Claude shifts to **read-only tools** (`read_file`, `grep_search`). It processes the codebase, maps dependencies, and writes an execution blueprint to your `.claude/plans/` directory.

```bash
# Ask Claude to formulate a blueprint without mutating the repository
claude "/mode plan Migrate our authentication layer from JWT to local Sessions"

```

## 11. Feature-Focused Workflows (Keeping Context in Control)

To maximize token efficiency and maintain precision, follow a feature-focused lifecycle:

1. **Targeted Ingestion:** Use `/add path/to/file` to explicitly load only the components relevant to the immediate task.
2. **Execute and Verify:** Let Claude perform its modifications and execute tests via Shell Mode.
3. **Drop Context Early:** Once your feature tests pass and your git state is clean, run `/drop path/to/file` or clean the environment with `/clear` before tackling the next piece of business.

## 12. Skills

**Skills** are shareable, custom capabilities that supercharge Claude's workflows. They utilize **progressive disclosure** (lazy-loading) to keep token costs down. Instead of stuffing massive prompts into every turn, Claude reads only the skill's *metadata* initially, loading the full instruction set only when invoked.

* **Structure:** Skills live in `SKILL.md` files within your workspace context or home configuration folder (`~/.claude/skills/`).

```markdown
<!-- ~/.claude/skills/review-pr.md -->
---
name: review-pr
description: Thoroughly audits an unmerged feature branch against core security compliance protocols.
---
# Instructions
1. Run `git diff main` to capture changed modules.
2. Analyze the diff explicitly looking for hardcoded credentials or open SQL injection surfaces.

```

## 13. Model Context Protocol (MCP)

The **Model Context Protocol (MCP)** acts as a universal adapter, connecting Claude Code directly to third-party databases, web APIs, enterprise tracking tools, or custom infrastructure.

* **Configuration:** Define your active servers inside your global configuration file (`~/.claude/config.json`).
* **Example Setup:**

```json
{
  "mcpServers": {
    "jira-tracker": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-jira", "--host", "your-company.atlassian.net"]
    }
  }
}

```

Once registered, Claude dynamically accesses tools like `mcp_jira-tracker_update_issue` right alongside its native file-editing capabilities.

## 14. Sub-Agents & Agent Teams

When tackling massive codebases or parallel engineering streams, a single context window isn't enough. Claude Code can spin up dedicated **Sub-Agents** to isolate tasks.

* **Architecture:** Sub-agents run completely clean, independent context windows. They are given specific tools and file access to run heavy analysis loops, returning a polished, high-level summary to the parent window.
* **Orchestration:** You can trigger built-in agents using the `@` handle, or let Claude autonomously spin up sub-agents to multi-task.

```bash
# Delegate an isolated codebase investigation to a sub-agent
@codebase_investigator map out every endpoint dependency tied to the billing engine

```
